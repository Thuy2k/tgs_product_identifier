 0. BỐI CẢNH & LÝ DO TẠO PLUGIN MỚI
================================================================================

  HỆ THỐNG: POS "Thế Giới Sữa" — 650+ shop WordPress Multisite.
  PLUGIN PHỤ THUỘC: tgs_shop_management (bắt buộc)

  PLUGIN CŨ (tgs_lot_generator — đã backup → tgs_lot_generator_old_backup):
    Flow cũ: Chọn SP → Chọn biến thể → Sinh mã → In dán
    VẤN ĐỀ:
      1. 20.000+ SKU × nhiều biến thể → in ra rồi mò dán = DỄ NHẦM
      2. Biến thể chỉ 1 variant_id đơn → không hỗ trợ kết hợp
         (VD: "Đỏ + Size M + 1-3 tháng" cần 3 biến thể cùng lúc)

  GIẢI PHÁP MỚI (plugin tgs_product_identifier):
    Bước 1: In mã trống (chỉ barcode, chưa gắn SP) → dán lên SP vật lý
    Bước 2: Quét mã → gắn vào SP + biến thể kết hợp (không mò, quét là biết)
    Biến thể kết hợp N:N qua bảng map + combo_hash

  TRẠNG THÁI MÃ (local_product_lot_is_active trên wp_global_product_lots):
    200  = Mã mới sinh (trống, chưa gắn SP) — MỚI từ plugin này
    100  = Đã sinh + gắn SP nhưng chưa xác nhận vào kho (từ plugin cũ)
      1  = Đã xác nhận, tồn kho (in stock)
      0  = Đã bán / đã xuất
     -1  = Hỏng / mất

  LEDGER TYPES (local_ledger_type trên wp_X_local_ledger):
    16  = Phiếu sinh mã (plugin cũ, vẫn giữ tương thích)
    30  = Phiếu sinh mã trống (MỚI — TGS_LEDGER_TYPE_BLANK_CODES)
    31  = Phiếu định danh SP (MỚI — TGS_LEDGER_TYPE_IDENTIFY)

  ── 1.3 CỘT MỚI THÊM VÀO wp_global_product_lots (qua migration):

  variant_combo_hash   VARCHAR(64) DEFAULT NULL
    → MD5 hash của sorted variant IDs: MD5("3,7,12")
    → Dùng để query nhanh tất cả lot cùng combo biến thể
    → INDEX: idx_variant_combo_hash

  identifier_ledger_id BIGINT DEFAULT NULL
    → FK → local_ledger.local_ledger_id (type=31)
    → Khi gắn mã vào khối, cột này = ledger_id của phiếu định danh
    → Khi gỡ mã → set NULL
    → INDEX: idx_identifier_ledger

  ── 1.4 variant_combo_hash HOẠT ĐỘNG NHƯ THẾ NÀO:

  Khi gắn biến thể [3, 7, 12] cho lot:
    1. sort → [3, 7, 12]
    2. join → "3,7,12"
    3. MD5 → "abc123def456..."
    4. Ghi vào lot.variant_combo_hash VÀ block.variant_combo_hash
  → Query: WHERE variant_combo_hash = 'abc123...' → lấy tất cả lot cùng combo

  ── 1.5 TẠO BẢNG & MIGRATION:

  Tất cả bảng & migration được quản lý TẬP TRUNG trong class-tgs-database.php (tgs_shop_management):
    • sql_global_lot_variant_map() → C12
    • sql_global_identify_blocks() → C13
    • Migration #24: add variant_combo_hash to wp_global_product_lots
    • Migration #25: add identifier_ledger_id to wp_global_product_lots
    • 2 file migration: database/migrations/2026_03_19_000100_... & 2026_03_19_000200_...

  → Khi cần tạo/cập nhật bảng: deactivate → reactivate plugin tgs_shop_management
  → Tất cả đều idempotent (IF NOT EXISTS / SHOW COLUMNS)

  ── 1.6 product_lot_meta JSON FORMAT:

  Khi sinh mã trống, ghi vào product_lot_meta:
    {"blank_ledger_id": 123, "generated_by": "tgs_product_identifier"}

  → Dùng LIKE pattern để fallback query khi JSON chưa ghi vào ledger
  → Helper: blank_ledger_like($id) → '%"blank_ledger_id":NNN,%'

 4. MÀN HÌNH 1: SINH MÃ TRỐNG
================================================================================

  URL: ?page=tgs-shop-management&view=idtf-blank-create
  View: admin-views/blank-codes/create.php
  JS: assets/js/blank-codes-create.js

  FLOW:
    1. Form: #blankCodeForm
       - #blankTitle — Tên phiếu (auto-fill: "Phiếu sinh mã #XXXX — dd/mm/yyyy HH:mm")
       - #blankQty — Số lượng (1–10.000)
           + Nút chọn nhanh: 50 | 100 | 200 | 500 | 1.000 | 5.000 (chips)
           + Ước tính giấy in: "~ X trang giấy" (auto-update)
       - #blankNote — Ghi chú
       - Email + ngày giờ auto-fill từ tgsIdtf
    2. Submit → #btnGenBlank → #genOverlay + progress bar
    3. BATCH: BATCH_SIZE = 200 → recursive runBatch()
       - Mỗi batch: POST tgs_idtf_generate_blank_codes
       - Truyền: offset, ledger_id (từ batch đầu), quantity, blog_id
       - Response: { offset, total, done, ledger_id }
       - Cập nhật #genProgressBar, #genProgressCount, #genProgressText
    4. Khi done=true → redirect tới detail page

  BACKEND:
    - Batch đầu (offset=0): tạo ledger type=30, sinh ticket_code SMT-XXXXXXXX
    - Mỗi mã: generate_ean13() → retry 5 lần nếu trùng
    - INSERT vào wp_global_product_lots: status=200, product_lot_meta JSON
    - Batch cuối (done=true): query tất cả lot theo blank_ledger_like()
      → ghi JSON lot_ids vào local_ledger.local_ledger_item_id

  LƯU Ý HIỆU NĂNG:
    - lot_ids CHỈ ghi 1 lần ở batch cuối (tránh O(n²) read-merge-write)
    - get_blank_lots() có fallback qua product_lot_meta LIKE nếu JSON chưa ghi

 11. QUY TẮC BẢO MẬT & HIỆU NĂNG
================================================================================

  BẢO MẬT:
    - Nonce verify ở TẤT CẢ endpoints (check_ajax_referer / wp_verify_nonce)
    - sanitize_text_field() cho mọi input text
    - intval() cho mọi ID / số
    - $wpdb->prepare() cho mọi query (chống SQL injection)
    - esc_js() cho output trong <script>
    - esc_attr() cho output trong HTML attributes
    - esc_url() cho URLs
    - jQuery .text() / esc() helper cho output XSS-safe
    - Transient cho print (TTL 5 phút, 1 lần dùng → xóa)

  HIỆU NĂNG:
    - Barcode collision: retry 5 lần (generate_ean13_raw + SELECT 1 LIMIT 1)
    - Batch INSERT IGNORE cho variant_map (1 query thay N×M)
    - JSON lot_ids chỉ ghi 1 lần ở batch cuối (tránh O(n²))
    - Fallback: product_lot_meta LIKE khi JSON chưa ghi
    - codes_count cache trên block (cập nhật mỗi assign/unassign)
    - Server-side pagination: 100/page (lots), 50/page (block codes), 20/page (list)
    - Print: < 200 → GET, >= 200 → transient (tránh URL limit 8KB)
    - Combo hash: MD5 lookup thay N-way JOIN
